# Access Control

В качестве модели управления доступом используется **мандатная модель** (Mandatory Access Control) по принципу *Bella-LaPadula*.
Документация:
- <https://wiki.astralinux.ru/pages/viewpage.action?pageId=27362553>

### Ключевые понятия

- **Конфиденциальность** (confidentiality): число (неотрицательное, 8-ми битное). Определяет уровень конфиденциальности субъекта или объекта.
    - *Чтение* разрешено, если конфиденциальность субъекта >=  конфиденциальности объекта
    - *Запись* разрешена, если конфиденциальность субъекта <=  конфиденциальности объекта
- **Категории (конфиденциальности)** (category): число (неотрицательное, 64-х битное). Определяет битовую маску (bitmask) категорий, в которые входит субъект или объект.
    - *Чтение* разрешено, если в категории субъекта включены все категории объекта
    - *Запись* разрешена, если в категории объекта включены все категории субъекта
- **Категории целостности** (integrity): число (неотрицательное, 32-х битное). Определяет 8-ми битную маску (bitmask) категорий целостности, в которые входит субъект или объект.
    - *Запись* разрешена, если в категории целостности субъекта входят все категории целостности объекта

## Формат ответа при отказе в доступе

При отказе в доступе прокси возвращает JSON-ответ с заголовком `Content-Type: application/json`.
Тело ответа содержит структурированную информацию о причине отказа:

```json
{
  "status": 403,
  "reason": "HOST_CONFIDENTIALITY_VIOLATION",
  "message": "MAC: read operation denied - user confidentiality range [1, 1] does not overlap with object range [2, 3]"
}
```

| Поле      | Тип    | Описание                                                                 |
|-----------|--------|--------------------------------------------------------------------------|
| `status`  | int    | HTTP-статус код ответа (401, 403 и т.д.)                                 |
| `reason`  | string | Машиночитаемый идентификатор причины отказа (см. таблицу ниже)           |
| `message` | string | Человекочитаемое описание причины отказа с деталями проверки              |

## Статусы отказа в доступе (Deny Reasons)

### Аутентификация

| Reason                      | HTTP Status | Описание                                                                                          |
|-----------------------------|-------------|---------------------------------------------------------------------------------------------------|
| `AUTHENTICATION_REQUIRED`   | 401         | Отсутствует или невалиден заголовок `Authorization: Negotiate`. Требуется Kerberos-аутентификация. |

### Контекст безопасности недоступен

| Reason                              | HTTP Status | Описание                                                                                                  |
|-------------------------------------|-------------|-----------------------------------------------------------------------------------------------------------|
| `USER_CONTEXT_UNAVAILABLE`          | 403         | Не удалось получить MAC-контекст пользователя из LDAP (пользователь не найден, отсутствуют атрибуты и т.д.) |
| `HOST_CONTEXT_UNAVAILABLE`          | 403         | Не удалось получить MAC-контекст целевого хоста из LDAP или кэша.                                          |
| `URI_CONTEXT_UNAVAILABLE`           | 403         | Не удалось получить MAC-контекст URI-правила из LDAP или кэша.                                             |

### Нарушение политики MAC на уровне хоста

| Reason                              | HTTP Status | Описание                                                                                                                                  |
|-------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------|
| `HOST_CONFIDENTIALITY_VIOLATION`    | 403         | Диапазон конфиденциальности пользователя не удовлетворяет требованиям хоста. Для чтения — диапазоны не пересекаются; для записи — не совпадают. |
| `HOST_CATEGORIES_VIOLATION`         | 403         | Категории безопасности пользователя не включают все категории, требуемые хостом.                                                            |
| `HOST_INTEGRITY_VIOLATION`          | 403         | Категории целостности пользователя не включают все категории целостности, требуемые хостом (проверяется только для операций записи).          |

### Нарушение политики MAC на уровне URI

| Reason                              | HTTP Status | Описание                                                                                                                                      |
|-------------------------------------|-------------|-----------------------------------------------------------------------------------------------------------------------------------------------|
| `URI_CONFIDENTIALITY_VIOLATION`     | 403         | Диапазон конфиденциальности пользователя не удовлетворяет требованиям URI-правила. Для чтения — диапазоны не пересекаются; для записи — не совпадают. |
| `URI_CATEGORIES_VIOLATION`          | 403         | Категории безопасности пользователя не включают все категории, требуемые URI-правилом.                                                          |
| `URI_INTEGRITY_VIOLATION`           | 403         | Категории целостности пользователя не включают все категории целостности, требуемые URI-правилом (проверяется только для операций записи).        |

### Транспортный уровень (L3/L4)

| Reason                                    | HTTP Status | Описание                                                                                                      |
|-------------------------------------------|-------------|---------------------------------------------------------------------------------------------------------------|
| `TRANSPORT_CONFIDENTIALITY_VIOLATION`     | 403         | Диапазон конфиденциальности хоста-источника не удовлетворяет требованиям хоста-назначения.                      |
| `TRANSPORT_CATEGORIES_VIOLATION`          | 403         | Категории безопасности хоста-источника не включают все категории, требуемые хостом-назначением.                  |
| `TRANSPORT_RESOLUTION_FAILED`             | 403         | Не удалось выполнить обратное DNS-разрешение IP-адреса в FQDN для транспортной авторизации.                     |
| `TRANSPORT_HOST_CONTEXT_UNAVAILABLE`      | 403         | Не удалось получить MAC-контекст хоста при транспортной авторизации.                                            |

### Прочие

| Reason            | HTTP Status | Описание                                                                  |
|-------------------|-------------|---------------------------------------------------------------------------|
| `BAD_REQUEST`     | 400         | Запрос некорректен (например, отсутствует заголовок `Host`).               |
| `INTERNAL_ERROR`  | 500         | Внутренняя ошибка сервера авторизации.                                    |

## Примеры ответов

### 401 — Требуется аутентификация

```http
HTTP/1.1 401 Unauthorized
Content-Type: application/json
WWW-Authenticate: Negotiate

{"status":401,"reason":"AUTHENTICATION_REQUIRED","message":"Kerberos authentication required"}
```

### 403 — Нарушение конфиденциальности на уровне хоста

```http
HTTP/1.1 403 Forbidden
Content-Type: application/json

{"status":403,"reason":"HOST_CONFIDENTIALITY_VIOLATION","message":"MAC: read operation denied - user confidentiality range [1, 1] does not overlap with object range [2, 3]"}
```

### 403 — Нарушение категорий на уровне URI

```http
HTTP/1.1 403 Forbidden
Content-Type: application/json

{"status":403,"reason":"URI_CATEGORIES_VIOLATION","message":"MAC: read operation denied - user categories max 0xf do not include all required object categories min 0xff"}
```

### 403 — Нарушение целостности на уровне хоста (операция записи)

```http
HTTP/1.1 403 Forbidden
Content-Type: application/json

{"status":403,"reason":"HOST_INTEGRITY_VIOLATION","message":"integrity: access denied - user integrity categories 0x10 do not include all required object integrity categories 0xff"}
```

### 403 — Нарушение конфиденциальности на транспортном уровне

```http
HTTP/1.1 403 Forbidden
Content-Type: application/json

{"status":403,"reason":"TRANSPORT_CONFIDENTIALITY_VIOLATION","message":"MAC: read operation denied - user confidentiality range [1, 1] does not overlap with object range [3, 3]"}
```
